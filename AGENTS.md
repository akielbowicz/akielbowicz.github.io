# Agent Instructions — akielbowicz.github.io

Personal portfolio and CV site for Sasha (Augusto) Kielbowicz.
Served at **ak.saxa.xyz** via GitHub Pages.

## Quick commands

```bash
mise run build      # build HTML site into dist/
mise run cv:pdf     # build styled PDFs → out/ + dist/cv/
mise run cv:txt     # build plain-text CVs + plain PDFs → out/ + dist/cv/
mise run cv:all     # all CV formats (pdf + txt)
mise run serve      # build + local HTTP server at http://localhost:8000
mise run clean      # remove dist/ and out/
```

Always run `mise run build` before `cv:pdf` or `cv:txt` — the PDF/text
pipeline reads from `dist/`.

## How the build works

1. `build.py` uses **pandoc** (managed by mise) to render each `.md`
   source through `templates/page.html5` into `dist/`.
2. `--pdf` flag: pandoc-rendered HTML → **weasyprint** (via `uvx`) →
   PDF. Copies to both `out/` and `dist/cv/`.
3. `--txt` flag: strips front-matter, image badges, language toggles,
   and the hidden AI-prompt div from CV markdown, then pandoc
   `--to plain --wrap=none` → `.txt`. Also renders a minimal-CSS HTML
   → weasyprint → plain PDF. Copies all to `out/` and `dist/cv/`.
4. `llms.txt` is copied verbatim from repo root to `dist/`.

## CV source files — editing rules

**`cv/en.md` and `cv/es.md` are the canonical sources.** All other CV
assets (`CV-Consultant.md`, `linkedin-*.md`) must be kept in sync
whenever AstraZeneca (or any other current role) bullets change.

Files to update together when changing current-role experience:
- `cv/en.md`
- `cv/es.md`
- `cv/CV-Consultant.md`
- `cv/linkedin-copy-paste-en.md`
- `cv/linkedin-copy-paste-es.md`
- `cv/linkedin-profile-consultant.md`

After editing CV sources, always rebuild:
```bash
mise run build && mise run cv:all
```

## Compliance rules — AstraZeneca

Experience descriptions for AstraZeneca **must not** name:
- Specific internal tools, APIs, or product names
- Vendor system names used in integrations
- Internal project codenames
- Specific vulnerability details or security findings

Use abstract, architecture-level language instead. See `cv/en.md`
AstraZeneca section for the approved framing. When in doubt, describe
the *pattern* (e.g. "OAuth 2.0 PKCE") not the *target system*.

## Blog posts

New posts go in `_posts/YYYY-MM-DD-slug.md` with front matter:
```yaml
---
title: "Post Title"
lang: en   # or es
---
```

The blog index at `blog/index.md` is auto-generated by `build.py` —
do not edit it manually.

## Styling notes

- Site font: **IBM Plex Mono** (monospace, loaded from Google Fonts)
- PDF font: **Source Serif 4** (serif, loaded from Google Fonts)
- Do not add inline styles to `.md` sources (except the existing
  hidden div in `cv/en.md` and `cv/es.md` — leave it in place for
  HTML output; it is stripped automatically for plain-text export)
- The `@media (max-width: 768px)` CSS warning in the build output is
  a known weasyprint limitation — harmless, do not attempt to fix

## CI

`.github/workflows/deploy.yml` runs on every push to `master`:
1. `apt-get install libpango-1.0-0 libpangoft2-1.0-0 libharfbuzz0b libfontconfig1`
2. `mise run build`
3. `mise run cv:all`
4. Upload `dist/` to GitHub Pages

The `deploy` job is gated on `build` and targets the `github-pages`
environment.
